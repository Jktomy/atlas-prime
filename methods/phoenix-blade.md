# Phoenix Blade and Aegis Break — Prime

Phoenix Blade is Athena executing an exact Sword herself under one bounded Jayson-authorized authority envelope. It is Athena's counterpart to Jayson wielding that Sword through Oathbringer.

Phoenix Blade uses the Sword's sealed mission, source locks, allowed operations, stop boundary, proof, and rollback contract. It does not invoke Thread Engine and it is not a normal repository engine, standing automation route, work surface, or source of permanence authority.

A Sword executed through Phoenix Blade may use the repository substrate defined by that Sword, including Fresh Clone First or an exact GitHub-native transaction, but the defining fact is that Athena is executing a Sword. Direct repository work that is not Sword execution is not automatically Phoenix Blade.

Aegis Break is Athena's bounded route-selection and route-construction method. When a normal route is obstructed or unavailable, Aegis Break may select or construct any safe equivalent route—including a direct GitHub-native blob, tree, commit, ref, and draft-pull-request transaction—while preserving the approved goal, operator, target, risk class, protected-data class, allowed operations, stop boundary, proof, and rollback.

Aegis Break is not hardwired to Phoenix Blade, Spear, Oathbringer, or any one repository substrate. It never removes Aegis, widens scope, grants authority, turns an alternate route into standing authority, or silently changes the permanence boundary.

Spear remains Athena's direct Thread Engine delivery route. Arrow and Bow remain Jayson/Artemis delegated delivery identities. Phoenix Blade and Aegis Break must not be used as aliases for either route.

Shardplate is the AI-assisted work surface. ChatGPT Work / Codex may be Shardplate, but Shardplate is never the mutation route or source of permanence authority.

Model provider, model, runtime, and Atlas runtime-control status are bound independently. Trusted reported OpenAI, Google, or Atlas-controlled local-model tokens map only to Spirallight, Chromelight, or Emberlight under `governance/investiture-source-identity-contract.md`. Unreported model use is `UNAVAILABLE`; deterministic non-model work has no Light and zero BEU. A Light does not identify an operator, work surface, credential principal, launcher, repository substrate, engine, route, or permanence authority.

For every invocation, bind the authorizer, operator, provider, model, runtime control, delivery route, launcher, repository engine, repository substrate, work surface, accounting Light or no-Light status, exact source, allowed operations, stop boundary, proof, and rollback independently.
