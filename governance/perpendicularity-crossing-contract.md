# Perpendicularity crossing contract

## Purpose

A Perpendicularity is Atlas's governed crossing protocol for moving an integrity-bound package or bounded retrieval request between otherwise separate realms. It is a protocol and authority boundary, not a synonym for one vendor, folder, device, API, or storage system.

Every crossing is mediated by Emberdark. A transport may deliver bytes or an event, but transport availability never grants intake, memory, source, infrastructure, recovery, READY, MERGE, or external-action authority.

## Approved architecture directions

- **Google Drive Perpendicularity** — the approved cloud-AI crossing for Athena-facing inbox/outbox exchange, sanitized Gemstones, bounded retrieval, and AI-to-AI communication.
- **iCloud-Forge working bridge** — a designated Atlas folder hydrated through iCloud for Windows on Apollo and synchronized two ways with a designated Forge mirror. It is a replaceable working-file bridge, not backup, recovery authority, or Coppermind authority.
- **Direct MCP/API crossing** — a future authenticated interface for Athena, Harmony, Glass Codex, VS Code, or another approved client to submit closed Emberdark requests or receive bounded results.
- **Offline crossing** — a sealed, integrity-verified package route for blocked connectors, recovery, or deliberate offline transfer.

No transport is activated merely because it is named here. Exact clients, folders, endpoints, credentials, network routes, and runtime facts remain protected implementation evidence.

## Closed crossing envelope

Every accepted crossing binds:

- crossing and request identity;
- source realm and destination realm;
- transport class;
- original request or immutable source reference;
- interpreted objective where an AI is involved;
- data classification and allowed destination classification;
- provenance and producing authority;
- integrity digest and declared payload inventory;
- requested authority and explicit denied actions;
- retention disposition;
- idempotency and replay identity;
- expiry when relevant;
- stop conditions and receipt identity.

The original Jayson request and an AI interpretation remain distinct fields. Valid syntax cannot silently redefine intent.

## Emberdark responsibilities

Emberdark validates schema, identity, classification, authorization, digest, inventory, replay status, destination eligibility, retention disposition, and stop conditions before routing. It may accept, reject, quarantine, request clarification, retain an approved Coppermind derivative, create a public-clean Mission, dispatch a bounded worker, or return a result.

Coppermind is not the mandatory transit path. Rejected payloads, temporary model output, partial files, and intermediate retries do not become durable memory by default.

## iCloud-Forge bridge boundary

The designated iCloud-Forge collection is a two-way synchronized current working set intended for seamless access from Jayson's approved Apple surfaces and local availability to Atlas through Forge.

- Apollo remains a replaceable, human-operated bridge endpoint and holds no unique authoritative state.
- Files must be locally hydrated before synchronization.
- Modification, rename, and deletion may propagate in both directions.
- Synchronization is not backup; Elantris owns independent backup, restore, and recovery proof.
- Emberdark processes a stable copy outside the live mirror and records provenance back to the synchronized source.
- Coppermind derivatives never silently overwrite the synchronized original.

## Public-clean export boundary

Google Drive, Gitea, GitHub, Phoenix, and cloud models receive only destination-eligible public-clean or explicitly sanitized packages. Protected originals and unrestricted private records do not cross merely because a connector can access them.

A private Gitea repository may later serve Phoenix after separate deployment and proof. GitHub remains canonical for Prime until an explicit approved cutover and may receive only deliberately prepared public or collaborative projects.

## Failure and recovery

Malformed, ambiguous, stale, replayed, partially synchronized, digest-mismatched, or classification-uncertain input is quarantined and fails closed. A blocked connector stops that route while preserving one resumable request identity. No blind retry creates a second Mission or duplicate external action.

Loss of Apollo pauses the iCloud bridge but does not destroy iCloud originals or Forge's last synchronized state. Loss of one Perpendicularity never blocks Prime recovery through approved canonical and recovery routes.

## Acceptance boundary

This contract defines source architecture only. It does not install a client, create a folder, activate MCP, deploy Emberdark, connect Google Drive or iCloud, import protected records, install Gitea, change GitHub authority, or prove backup and restore.
