# Prime Source Lifecycle

Prime is the sole active canonical authored source. A source file controls only the scope declared by its metadata and routing. Tests, generated indexes, receipts, previews, issue text, chat, runtime state, and predecessor repositories are evidence or projections, not competing authority.

`governance/repository-process-contract.md` governs the complete-candidate, route, validation, review, repair, permanence, and readback mechanics for durable repository change.

## Lifecycle

1. **Capture** clean, durable meaning without protected values.
2. **Classify** it as identity, governance, project, operation, quest, protocol, runbook, template, infrastructure source, generator, or evidence.
3. **Route** every canonical file from an active command or registry surface.
4. **Lock** the exact canonical repository, main head, controlling source, and transaction identity.
5. **Discover** the complete deterministically knowable source, continuity, recovery, schema, test, and generated consequences.
6. **Author** one prepublication-complete candidate with exact paths, bytes, hashes, tree, stop point, and rollback.
7. **Deliver** through Spear or immutable Arrow/Bow to Thread Engine, or through an exact Sword/Oathbringer alternate route when authorized.
8. **Validate and review** through the mandatory baseline, affected invariants, required platforms, actionable review reconciliation, and zero or one normal consolidated repair.
9. **Strikeforce** the unchanged exact candidate head through Noctua, Ares, and Aegis.
10. **Ready** only the unchanged validated head. Without Shardblade authority, report `Prime PR #___ is ready to merge.`
11. **Merge** manually by Jayson by default, or through one exact transaction-scoped Shardblade action when the current Jayson instruction explicitly invokes it.
12. **Read back** the merged PR, merge SHA, canonical main, tree, paths, continuity, generated-state consequence, receipts, and rollback posture.
13. **Diagnose** nonauthoritative projections in temporary storage as part of ordinary validation.
14. **Sunset** superseded source with an explicit replacement or archive pointer.

## Source boundaries

- Active source must be necessary, current, nonduplicative, and Prime-native.
- Historical, failed, preview, harvest, closeout, release, and predecessor-only material stays in the frozen predecessor or the final Receipt Gemstone unless a current canonical contract explicitly requires a sanitized record.
- Private evidence stays in its approved evidence system; Prime may keep only sanitized policy and pointers.
- A source removal requires a disposition: KEEP, MERGE, REMODEL, REGENERATE, ARCHIVE, or EXCLUDE.
- Projection diagnostics are temporary, carry their source fingerprint, and are reproducible from merged Prime source without committed report files.
- A partial candidate is not made acceptable by calling the PR a draft; deterministically knowable dependencies belong in the initial candidate.

## Change routes

- Ordinary source: Athena Weave → Spear → Thread Engine.
- Immutable carrier: Athena Weave → Arrow → Bow → Thread Engine.
- Protected non-self-change: exact Aegis Break authority → Thread Engine protected route or another exact approved route.
- Thread Engine self-change or independent emergency bootstrap: Aegis Break → Oathbringer.
- Projection diagnostics: read-only temporary evaluation inside ordinary validation; no follow-up publication transaction.
- Route failover: read-only reconciliation of the same transaction before any alternate publisher acts; never create a duplicate mission, branch, or PR.

## Permanence authority

A direct Jayson instruction to change, repair, update, add to, or Sunset Prime authorizes one bounded transaction through merge-ready. It includes complete-candidate construction, candidate-caused repair, actionable review repair, validation, exact-head Strikeforce, and the ready-for-review transition without a second user command. Preview-only and draft-only instructions remain narrower.

`with Goddess Mode` authorizes bounded autonomous completion only inside that transaction and its safeguards. It never grants permanence.

The default route does not include machine merge. Jayson manually merges after the unchanged head is ready. A current explicit `with Shardblade` instruction or unambiguous equivalent authorizes one exact-head compare-and-swap merge after required statuses, review reconciliation, Strikeforce GREEN, replay reservation, rollback proof, and fresh readback. That one-use authority is consumed by success or terminal safe rejection.

If candidate bytes change after READY, return the PR to draft, revalidate and review the replacement exact head, repeat Strikeforce, and use exact current permanence authority for that replacement. Post-merge correction uses a new reviewed repair-forward or revert PR; history rewrite is forbidden.

The frozen `Jktomy/atlas-codex` repository is predecessor and rollback evidence only. It is never an active source, runtime dependency, recovery dependency, or future update destination after cutover.
