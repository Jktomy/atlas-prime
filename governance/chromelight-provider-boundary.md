# Chromelight provider and account boundary

## Accounting-name separation

This RP-C03 proper name and gate preserve the accepted provider/account lineage.
They are not token accounting evidence. Under
`governance/investiture-source-identity-contract.md`, accounting Chromelight is
created only from trusted provider/runtime-reported Google model tokens. Public
eligibility research, a Google product label, or the legacy v1 `stormlight`
field neither emits Chromelight nor activates a provider, account, repository,
model, runtime, capability, or route.

RP-C03 reconciles public evidence about Google eligibility, the Atlas Gemini
Scout role, Jules, and Antigravity Shardplate. Evidence reconciliation is not
provider enrollment, account acceptance, installation, authentication,
repository authorization, task launch, or capability promotion.

## Identity separation

`Gemini Scout` is an Atlas research role, not the name of a separately verified
Google product. A possible Google surface must be named independently. Gemini
Code Assist agent mode is documented as a preview IDE pair-programming surface
with file, terminal, and optional MCP tools. Its provider identity does not make
it an Atlas operator, repository engine, mutation route, or source of authority.

Jules is a hosted coding agent with its own Google-account and GitHub-application
boundaries. Official documentation requires sign-in, acceptance of a privacy
notice, and explicit selection of repositories before it can work on them.
Those steps are external authorization actions and were not performed here.

Antigravity is an agentic development platform available on Windows, macOS, and
Linux. It can act across editor, terminal, and browser surfaces. In Atlas it may
be evaluated only as Shardplate: an AI-assisted work surface. Shardplate is
never the delivery route, repository engine, credential principal, or source of
permanence authority.

## Account and data boundary

Public documentation can establish product prerequisites and documented
availability, but it cannot establish Jayson's account type, age eligibility,
region, subscription, administrator policy, accepted terms, installed state,
or current quota. Those facts require an explicit human-controlled account
readback. No login, consent, subscription, installation, GitHub application
grant, repository selection, or provider activation is authorized by this gate.

Private repository content, credentials, tokens, local browser state, and
private data remain outside public research. Any later provider trial requires
a new exact agentic warrant, the provider-activation approval boundary, least
repository access, a non-production harmless scope, receipts, and separate
permanence review.

## Evidence and disposition

The machine-readable source and disposition ledger is
`proof/repairing-prime/rp-c03-chromelight-evidence-r01.json`, validated against
`schemas/chromelight-evidence-register-v1.schema.json`. It records official
Google documentation observed on 2026-07-12, identifies every unverified
operator fact, and keeps every provider surface `NOT_ACTIVATED`.

`CHROMELIGHT_EVIDENCE_RECONCILED` means the evidence and blockers are explicit.
It does not prove provider eligibility, activate Gemini Scout, grant Jules
repository access, install Antigravity, promote a capability, or advance an AJ.
