# Athena Routes

This module preserves the hosted Jayson/Artemis Arrow/Bow boundary, exact mission
compilation, and retained recovery components. The directory name is a legacy
implementation namespace; it does not make Arrow/Bow an Athena method.

```text
Athena Spear -> Thread Engine
Athena Sword -> Phoenix Blade -> GitHub transaction, no Thread Engine
Athena Aegis Break -> direct GitHub-native or other bounded safe route
Jayson Arrow -> Artemis Bow -> Thread Engine
Jayson Sword -> Oathbringer -> GitHub transaction, no Thread Engine
```

## Hosted Arrow/Bow intake

The hosted route accepts one locally pre-screened, public-clean, immutable Arrow
carrier from an owner-authorized event, binds trusted GitHub run identity,
rejects replay and protected paths, invokes the existing Spear compiler, and
passes the compiler output unchanged to the singular Prime Thread Engine.

This is Jayson and Artemis delegated delivery. It is not Athena's Spear route.

The hosted intake contains no independent git tree, commit, push, or pull-request
implementation. It has no direct-main, force-push, ready, merge, settings,
cleanup, standing credential, or second-writer authority. Thread Engine
self-change is handed to Aegis Break and an independent method before mutation.

The hosted result stops at draft-PR readback. A `PARTIAL` Thread Engine result is
preserved exactly and blocks retry. Generated projections and protected paths
are not accepted by the ordinary hosted route.

## Free-form and guided publisher components

`tools.athena_routes.free_form_intake` and
`tools.athena_routes.guided_publisher` remain implementation and emergency
recovery components for preparing or dispatching an owner-authorized Arrow/Bow
carrier.

They are not routine user interfaces. Jayson is not required to run Python,
PowerShell, a custom gateway, an OAuth server, or a separate GitHub App to
authorize Athena. Jayson's explicit in-chat authorization of the exact Preview
or mission is sufficient human authority.

These components preserve exact canonical-main and workflow-source binding,
immutable carrier and mission identity, privacy screening, deterministic branch
and replay rejection, durable no-retry intent, owner-session and hosted actor
readback, partial-state preservation, singular Thread Engine execution,
immutable branch and draft-PR stop, and no automatic ready or merge.

## Direct Spear

Athena's direct Spear route is defined by `methods/athenas-spear.md`. Spear
delivers Athena's exact Weave into the singular Prime Thread Engine. Accepted
live evidence is PR #39 with validation run `29056870868`, corroborated by PR
#102 with validation run `29215320609`.

Spear is not the owner-guided Arrow/Bow publisher. Byte equivalence does not
erase operator or route identity.

## Retired fresh-origin experiment

The former `fresh_work_bridge` construction and platform-attestation premise are
retired. They are not active route requirements, capability gates, blockers, or
user actions. Git history preserves that experiment; current source does not
depend on it.

## RP-C01-M05 parity join

The M05 parity helper binds the direct compile receipt and hosted Thread Engine
evidence from one exact carrier. Its record is evidence only. It cannot execute,
retry, merge, confer permanence, promote a capability or Quest, or create a
second writer.
